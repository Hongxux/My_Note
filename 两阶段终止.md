
- 含义：在一个线程 T1 中“优雅”地终止线程 T2?
	- 优雅：指的是给 T2 一个料理后事的机会。
- 实现方式一：使用打断标记实现
	- 工作线程：![[Pasted image 20251130154352.png]]
		```
		@Override
		public void run() {
			try {
				while (!interrupted()) { // 检查终止标志位
					try {
						// 模拟线程的工作内容，可能包含阻塞操作
						Thread.sleep(1000);
						System.out.println("Working...");
					} catch (InterruptedException e) {
						// 捕获中断异常，通常选择重新设置中断标志并退出循环
						Thread.currentThread().interrupt(); // 重新设置中断标志
						break; // 退出工作循环
					}
				}
			} finally {
				// 清理阶段：释放资源，如关闭文件、网络连接、释放锁等
				doCleanup();
				System.out.println("Thread terminated gracefully.");
			}
		}
		
		private void doCleanup() {
			// 执行必要的清理工作
		}
		```
		- 保证中断信号的传递性：
			- 实现方式：在捕获 `InterruptedException`后，通常需要调用 `Thread.currentThread().interrupt()`来**重新设置线程的中断状态**。
			- 原因：当异常被抛出时，中断状态会被清除。
		- finally块：将清理逻辑放在 `finally`块中
			- 清理：对系统资源的释放
			- 保证无论线程是正常退出循环还是因异常跳出，清理工作都会被执行
