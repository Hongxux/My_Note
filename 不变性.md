**易混点**：在“**不可变的对象**”与“**不可变的对象引用**”之间存在着差异。保存在不可变对象中的程序状态仍然可以更新，即通过将一个保存新状态的实例来“替换”原有的不可变对象。
- **不可变的对象（Immutable Object）​**​：指的是对象一旦创建，其状态（即数据）就不能被修改。**所有域为`final`，且不泄露内部引用**。例如，Java中的String类是不可变的。如果你有一个String对象，你不能改变它的字符序列；任何修改都会-创建一个新的String对象。
- ​**不可变的对象引用（Immutable Object Reference）​**​：这通常指的是引用（如变量）本身是不可变的，意味着你不能改变这个引用指向另一个对象。例如，在Java中，用`final`关键字声明的变量：`final String s = "hello";`，你不能将`s`重新赋值为另一个String对象。

----
**[[构建不可变对象指南]]**
满足以下条件，对象才是不可变的：
        
1. 创建后其状态不能改变。不提供修改内部状态的方法，且返回可变对象时需防御性复制（如`Arrays.copyOf()`）
	
2. 所有域都是`final`类型。`final`关键字除了表示不可变，还能确保**初始化安全性**——即被正确构造的不可变对象，无需同步就能被所有线程安全地访问。
	
3. 对象是正确创建的（在构造函数期间，`this`引用没有逸出）。
