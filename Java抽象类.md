抽象类是一种特殊的类，用于在继承层次中定义通用模板，它允许包含抽象方法（无实现）和具体方法（有实现），但不能直接实例化。

**抽象类的定义**​：使用`abstract`关键字声明抽象类（如`public abstract class Person`）。
- **抽象类的包含：**
	- 抽象类可以包含抽象方法（如`public abstract String getDescription();`），这些方法没有实现，充当占位符，**强制子类提供具体实现**。
	- 抽象类也可以包含**具体方法（如`getName()`）、字段和构造函数**。
	- **设计原则**​：抽象类用于建模“is-a”关系，其中超类表示通用概念，子类提供具体实现。优先将公共字段和方法移至超类（无论是否抽象）。
- **实例化限制**​：抽象类不能直接实例化（如`new Person("Vince Vu")`会编译错误），但可以声明变量引用具体子类的对象（如`Person p = new Student("Vince Vu", "Economics");`）。
- **设计优势**​：抽象类允许将通用功能（如`getName`）提升到超类，减少代码重复；同时通过抽象方法强制子类实现特定行为，确保一致性。

**子类责任**​：子类继承抽象类时，必须实现**所有**抽象方法，否则子类也必须声明为抽象类（实现部分的也要生声明为抽象类）。例如，`Student`类实现了`getDescription`方法，因此它是具体类。
