- 需求背景：**双亲委派模型的局限性**
	- Java 核心库中定义的接口（Service Provider Interface, SPI），无法直接加载由第三方厂商在 classpath 下提供的实现
		- 父加载器无法调用子加载器的类
- 解决措施：线程上下文类加载器
	- ​ 当 `DriverManager`（由启动类加载器加载）需要加载数据库驱动时，它不再遵循双亲委派，而是**获取当前线程的上下文类加载器（默认是AppClassLoader）**，然后用这个类加载器去加载并实例化 classpath 下的驱动实现类
- 获取方式：通过 `Thread.currentThread().getContextClassLoader()`获取
- 设置方式：通过 `Thread.currentThread().setContextClassLoader()`设置
	- 如果没有手动设置，线程会继承其父线程的上下文类加载器。Java 应用初始线程的上下文类加载器默认就是**应用程序类加载器**
