当一个方法需要被多个切面增强时（例如，先记录日志，再管理事务），就需要确定切面的执行顺序。图2给出了解决方案。

- **使用 `@Order`注解**：在切面类上使用 `@Order`注解来控制顺序。
    
- **排序规则**：为 `@Order`注解的 `value`属性指定一个整数值。**数字越小，优先级越高**，该切面的通知方法越先执行。
    

**示例**：

```
@Component
@Aspect
@Order(1) // 数字小，先执行：第一个切面（如日志）
public class LogAspect {
    // ...
}

@Component
@Aspect
@Order(2) // 数字大，后执行：第二个切面（如事务）
public class TxAspect {
    // ...
}
```
