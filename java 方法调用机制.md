- 静态绑定：所有`private`、`static`、`final`方法以及**方法重载（Overloading）**
	- 发生时期：编译时
	- 字节码指令：
		- invokestatic：静态方法调用
		- invokespecial:构造方法、私有方法、父类方法
	- 含义：编译器能够仅凭变量的**声明类型**就确定应该调用哪个方法。
		- 这些方法都是子类无法继承的，无需运行的时候判断变量的具体类型
- 动态绑定：public, protected(可重写的方法)
	- 发生时期：运行时
	- 字节码指令：invokevirtual，invokeinterface
		- invokevirtual：实例方法（可重写的方法）
		- invokeinterface：接口方法调用
	- 含义：运行时根据对象的**实际类型**决定调用哪个方法。**支持多态**
		- 这些方法都是子类可以继承的，需运行的时候判断变量的具体类型
	- 运行机制：
		1. 获取声明引用实际指向的对象（实际类型）
		2. 访问实际类型对象的类信息，找到其虚方法表
		3. 在虚方法表中查找对应方法的索引位置
		4. 根据索引找到方法地址
		5. 调用该方法
- **编译时重载解析**​：
	1. 编译器首先根据对象的声明类型和方法名找到所有可能的方法候选（包括超类中的可访问方法）
	2. 然后根据参数类型选择最佳匹配。
		- 方法签名（方法名和参数类型）是关键
		- 但返回类型不是签名的一部分。
