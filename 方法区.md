- 含义：概念上是一个新的内存区域，逻辑上是堆的一部分，具体实现不一样![[Pasted image 20251205152212.png]]
		 - JDK8以前，HotSpot是将其作为一个永生代，是堆内存的一部分
		- JDK8以后，将其变成元空间，是操作系统的内存一部分
	- 创建时间：运行时创建
	- 特点：
		- 线程间共享的
	- 异常：方法区内存溢出
		- 当JVM加载的类信息容量超过了方法区（元空间）的最大限制时，就会抛出 `OutOfMemoryError: Metaspace`
	- 内容：
		- 类型信息：来自于加载的类文件
			- 类的身份信息：对每个加载的类、接口、枚举、注解
				- 全限定名（如 `java.lang.String`）
				- 直接父类的全限定名
				- 实现的接口的有序列表
				- 修饰符（如 `public`, `abstract`）
			- 域（Field）信息：类的字段信息，并记录声明的顺序
				- 字段名称
				- 类型（如 `int`, `String`）
				- 修饰符（如 `private`, `static`）
			- 方法信息：`abstract`和 `native`方法除外，包括构造方法
				- 方法名
				- 返回类型
				- 参数列表（数量和类型）
				- 修饰符
				- 方法的字节码指令
				- 操作数栈的深度
				- 局部变量的大小
				- 异常表
		- 运行时常量池：
			- 含义：.class类文件的常量池，当该类信息被加载，（从磁盘到内存）
				- 它的常量值就会被放入运行时常量池（一块内存中）
				- 并且将里面的符号地址（#1等等），变为真实地址
			- 常量池：
				- 底层实现：一个固定大小的哈希表
				- 内容：编译期生成的![[Pasted image 20251207093656.png]] 
					- **字面量**（如直接写的数字、字符串）[[字符串常量池]]
					- **符号引用**（对类、字段、方法的引用
				- 作用：
					- 在虚拟机指令中调用![[Pasted image 20251207093558.png]]
	
		- **静态变量**：即被 `static`修饰的类变量。静态变量和类关联在一起，随着类的加载而加载。
			- 如果静态变量是引用类型（如 `static Object obj;`）
				- 变量本身（引用地址）存储在方法区，
				- 它所指向的实际对象实例通常存储在堆中
		  - **JIT编译后的代码**：为了提高效率，JVM的即时编译器（Just-In-Time Compiler）会将频繁执行的字节码编译成本地机器代码，这些编译后的代码缓存也存储在方法区
		  - **类加载器引用**：指向加载这个类的类加载器的引用
		  - **Class对象引用**：指向该类的 `java.lang.Class`对象的引用，这是Java反射机制的基石
