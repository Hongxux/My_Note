对象变量（如`Date rightNow`）不直接包含对象，而是指向堆内存中的对象。修改引用不会影响原对象。

1. **在Java中，对象变量直接存储对象本身。​**​
    
    - ​**答案**​：False
        
    - ​**理由**​：对象变量存储的是对象的引用（地址），而不是对象本身。例如，`Date rightNow = new Date();`中，rightNow存储的是指向Date对象的引用。
        
    
2. ​**使用new操作符创建对象时，构造器可以省略。​**​
    
    - ​**答案**​：False
        
    - ​**理由**​：构造器是创建对象的必要部分，new操作符必须调用构造器（如`new Date()`）。如果类没有显式构造器，Java会提供默认无参构造器，但构造器不可省略。构造器是一个特殊的函数，函数名与对象的名字一样。
        
    
3. ​**Java的对象变量可以设置为null，这表示变量未引用任何对象。​**​
    
    - ​**答案**​：True
        
    - ​**理由**​：null是特殊值，表示对象变量未指向有效对象。例如，`Date startTime = null;`后，调用startTime的方法会抛出异常。
        
    
4. ​**Java的对象引用机制与C++的对象指针完全相同。​**​
    
    - ​**答案**​：False
        
    - ​**理由**​：虽然相似，但Java引用更安全：不支持指针算术，且自动垃圾回收。C++指针允许更灵活的操作（如地址计算），而Java引用是受控的。
        
    
5. ​**如果对象变量未初始化，Java会自动将其设为null。​**​
    
    - ​**答案**​：False
        
    - ​**理由**​：局部对象变量不会自动初始化，必须显式赋值（如设为null或new对象）。否则编译时会报错。只有实例字段在未初始化时默认为null。
