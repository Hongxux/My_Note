#### 一、列表（List）的子范围

​**操作方式**​：

使用 `subList`方法提取列表的指定索引范围：

```
List<Employee> group2 = staff.subList(10, 20); // 提取索引 10（含）到 19（含）的元素
```

​**关键特性**​：

1. ​**索引规则**​：起始索引包含，结束索引不包含（与 `String.substring`规则一致）。
    
2. ​**视图特性**​：对子范围的操作会**直接影响原集合**。
    
    ```
    group2.clear(); // 删除子范围元素，原列表 staff 中对应元素同步被清除
    ```

#### 二、有序集合与映射的子范围（SortedSet/SortedMap）

​**操作逻辑**​：

基于元素的**自然排序**​（而非索引位置）定义子范围。

​**SortedSet 方法**​：

```
SortedSet<E> subSet(E from, E to);  // [from, to)  
SortedSet<E> headSet(E to);         // (-∞, to)  
SortedSet<E> tailSet(E from);       // [from, +∞)
```

​**SortedMap 方法**​：

```
SortedMap<K, V> subMap(K from, K to);  // [from, to)  
SortedMap<K, V> headMap(K to);         // (-∞, to)  
SortedMap<K, V> tailMap(K from);       // [from, +∞)
```

​**注意**​：返回的子集/子映射均为**视图**，修改会同步到原集合。

#### 三、导航集合的精确控制（NavigableSet - Java 6+）

​**增强功能**​：

通过布尔参数**显式指定边界是否包含**。

​**方法示例**​：

```
// 精确控制边界包含性  
NavigableSet<E> subSet(E from, boolean fromInclusive, E to, boolean toInclusive);  
NavigableSet<E> headSet(E to, boolean toInclusive);  
NavigableSet<E> tailSet(E from, boolean fromInclusive);
```

​**典型场景**​：

需严格定义包含/排除边界时（如区间查询）。

### 重点知识分层

|​**学习者层级**​|​**必须掌握的核心内容**​|
|---|---|
|​**初学者**​|`List.subList`的索引范围规则及其视图特性（操作同步影响原集合）。|
|​**进阶者**​|`SortedSet`/`SortedMap`基于值范围的子集操作，理解有序集合的范围定义逻辑。|
|​**高级应用**​|`NavigableSet`的边界精确控制，灵活处理包含/排除边界的需求。|
