- Bean的定位：由 Spring IoC 容器进行全生命周期管理的 Java 对象实例，是构成 Spring 应用的基本单元。
	- IoC容器管理Bean的方式：[[Bean的生命周期]]
- 问题：
	- 循环依赖
		- 通过**三级缓存**机制解决**单例 Bean 的 setter/字段注入**的循环依赖。
		- 优先使用构造器注入
			- 它能在应用启动时立即暴露循环依赖问题，促使你改进设计。良好的设计应避免循环依赖。
    - [[Bean 的作用域]]选择不当：误将应为原型作用域的 Bean 声明为单例
	    - 无状态的服务类通常用 `singleton`（默认）
	    - 有状态的 Bean 应使用 `prototype`
