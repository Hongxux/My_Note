当多个通知需要使用相同的切点表达式时，重复编写既麻烦又容易出错。图3展示了如何重用切点表达式。

- **使用 `@Pointcut`注解**：定义一个空方法，并用 `@Pointcut`注解标注，注解的 `value`属性为切点表达式。
    
- **引用方式**：在其他通知注解中，直接通过**方法名()**来引用这个切点定义。
    

**示例**：

```
@Component
@Aspect
public class MyAspect {

    // 1. 定义可重用的切点
    @Pointcut("execution(* com.example.service..*(..))")
    public void globalPointcut() {} // 方法体为空，仅作为标记

    // 2. 在多个通知中引用
    @Before("globalPointcut()") // 引用切点
    public void beforeAdvice() {
        System.out.println("前置通知");
    }

    @AfterReturning("globalPointcut()") // 再次引用同一个切点
    public void afterAdvice() {
        System.out.println("后置通知");
    }
}
```

这种方法极大地提高了代码的**可维护性**。如果需要修改切点，只需修改 `@Pointcut`注解一处即可。
