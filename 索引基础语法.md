

### 1. 创建索引

​**用于在已有的表上为指定的列创建索引，以加快查询速度。​**​

```
CREATE [UNIQUE|FULLTEXT] INDEX index_name ON table_name (index_col_name, ...);
```

- ​**`[UNIQUE | FULLTEXT]`**​ （可选关键字）：
    
    - ​**`UNIQUE`**​：创建唯一索引，要求索引列的值必须是唯一的。
        
    - ​**`FULLTEXT`**​：创建全文索引，用于全文搜索，适用于大文本字段。
        
    - 如果省略，则创建的是普通的常规索引。
        
    
- ​**`index_name`**​：为要创建的索引取一个名称，通常以 `idx_`开头以便识别。
    
- ​**`table_name`**​：要在哪个表上创建索引。
    
- ​**`(index_col_name, ...)`**​：指定一个或多个列名。为多个列创建索引称为**复合索引**。
    

​**示例**​：

```
-- 在 user 表的 name 列上创建一个名为 idx_user_name 的普通索引
CREATE INDEX idx_user_name ON user (name);

-- 在 user 表的 email 列上创建唯一索引，确保邮箱不重复
CREATE UNIQUE INDEX idx_unique_email ON user (email);

-- 在 article 表的 title 和 content 列上创建全文索引
CREATE FULLTEXT INDEX idx_ft_title_content ON article (title, content);

-- 在 order 表上创建基于 user_id 和 create_time 的复合索引
CREATE INDEX idx_user_ordertime ON `order` (user_id, create_time);
```

### 2. 查看索引

​**用于显示指定表上存在的所有索引的信息。​**​

```
SHOW INDEX FROM table_name;
```

- 执行此命令后，会返回一个结果集，包含索引名称、索引类型、索引所在的列、是否唯一等详细信息。这是检查索引是否创建成功或分析表索引结构的必备命令。
    

​**示例**​：

```
-- 查看 user 表的所有索引
SHOW INDEX FROM user;
```

### 3. 删除索引

​**用于删除表中已存在的索引。​**​

```
DROP INDEX index_name ON table_name;
```

- ​**`index_name`**​：要删除的索引的名称。
    
- ​**`table_name`**​：该索引所在的表名。
    

​**示例**​：

```
-- 删除 user 表上名为 idx_user_name 的索引
DROP INDEX idx_user_name ON user;
```

### 核心要点总结

- ​**创建索引**​ 是为了**提高查询性能**。
    
- ​**查看索引**​ 是为了**验证和诊断**索引情况。
    
- ​**删除索引**​ 通常是因为索引不再需要，或者为了**优化数据写入性能**​（因为索引会降低插入、更新、删除的速度）。
    

这三个命令是数据库管理和性能优化中最基本、最常用的操作。
