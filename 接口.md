

接口是Java编程语言中的一种引用类型，它定义了一组方法要求的契约，而不提供具体的实现。任何类可以通过实现接口来承诺提供接口中定义的所有方法。
1. ​**接口的契约性质**​：接口定义了"能做什么"，而不关心"怎么做"
    
2. ​**类型安全**​：接口为强类型语言提供了编译时检查机制
    
3. ​**多态支持**​：接口是实现多态的重要基础

#### 1. 接口静态方法（Java 8引入）

- ​**核心目的**​：消除工具类（Utility Classes）的必要性
    
- ​**语法特征**​：
    
    ```
    public interface Path {
        // 静态工厂方法
        public static Path of(URI uri) { ... }
    
        // 静态工具方法
        public static boolean isValid(String path) { ... }
    }
    ```
    
- ​**关键优势**​：
    
    - API内聚性：相关方法直接归属接口
        
    - 使用便捷性：无需记忆额外工具类名
        
    - 设计简洁性：减少类数量，消除工具类（Utility Classes）的必要性
        
    

#### 2. 接口私有方法（Java 9引入）

- ​**核心目的**​：接口内部代码复用（主要是default方法）
    
- ​**语法特征**​：
    
    ```
    public interface Formatter {
        default String format(Object obj) {
            String content = prepareContent(obj);
            return applyStyle(content);
        }
    
        // 私有实例方法
        private String prepareContent(Object obj) { ... }
    
        // 私有静态方法
        private static String applyStyle(String content) { ... }
    }
    ```
    
- ​**类型支持**​：
    
| 方法类型   | 可调用者             |
| ------ | ---------------- |
| 私有实例方法 | 默认方法、其他私有实例方法    |
| 私有静态方法 | 静态方法、默认方法、其他私有方法 |
|        |                  |

# 面试官关心的方面及答案

## 问题：接口和抽象类有什么区别？

​**答案：​**==接口不是类，而是对符合接口要求的类的**一组需求**==

**混合使用模式**​：抽象类提供基础实现，接口定义扩展点​
1. ​**抽象类 = 基础模板（状态+行为）**，通过归纳了子类共有的方法和字段，定义对象的本质，共享代码使用，使得子类能在父类的基础上进行修改和增强。​
    
2. ​**接口 = 能力契约**，解决"能做什么"问题（has-a）​，扩展了对象的能力。拥有多重继承的最大部分优点，缺避免了多重继承的复杂性问题。
    
    - 满足JAVA的强类型：编译器验证方法存在性
        
    - 多态支持：`Flyable[] flyingObjects = {new Bird(), new Drone()}`
        
    - 突破单继承限制：`class Bird extends Animal implements Flyable, Comparable<Bird>`
    

## 问题：为什么Java要引入接口概念？

​**答案：​**​

1. ​**实现多重继承的效果**​：解决Java单继承的限制
    
2. ​**定义行为契约**​：明确规范类应该提供什么功能
    
3. ​**实现多态**​：允许不同的类以统一的方式被处理
    
4. ​**降低耦合**​：基于接口编程而非具体实现编程
    
5. ​**支持API设计**​：为类库提供标准化的扩展点
    

## 问题：Java 8之后接口有哪些新特性？

​**答案：​**​

1. ​**默认方法**​：使用default关键字提供方法默认实现
    
2. ​**静态方法**​：接口中可以定义静态方法
    
3. ​**函数式接口**​：只有一个抽象方法的接口，支持lambda表达式
    
4. ​**私有方法**​：Java 9开始支持接口中的私有方法
    

这些特性使得接口更加灵活，既保持了契约的定义，又能在一定程度上提供代码复用。
