### 一、 MVC 是什么？

MVC 是一种**软件架构模式**（或称设计思想），它将一个应用程序的逻辑划分为三个核心组成部分，以实现**职责分离**。这种思想并非 Java 独有，而是广泛应用于各种语言和框架的 Web 开发中。

### 二、 三大组件的职责（分工协作）

图片中清晰地指出了每一部分的专注点：

| 组件                  | 职责          | 具体工作                                                                                          |
| ------------------- | ----------- | --------------------------------------------------------------------------------------------- |
| **Model（模型）**       | **业务处理与数据** | 负责核心业务逻辑(**Service**)（如用户登录验证、订单计算）、数据存取（**dao**）（与数据库交互）。它不关心数据如何展示。                         |
| **View（视图）**        | **数据展示**    | 负责将 Model 处理后的结果以特定形式（HTML 页面、JSON 数据、PDF 报表）呈现给用户。它不包含复杂的业务逻辑。                               |
| **Controller（控制器）** | **请求调度**    | 作为总调度中心<br>1.接收用户请求<br>2.决定调用哪个 Model 进行处理<br>3.选择哪个 View 来展示结果。<br>它是 Model 和 View 之间的“协调员”。 |

### 三、 为什么使用 MVC？—— 核心优点

其根本价值在于**解耦**。

1. **低耦合，扩展性强**
    
    - **解读**：三个组件各司其职，修改其中一个（如更换前端页面从 JSP 为 Thymeleaf，即 View），通常不会影响其他两个（Model 和 Controller）的逻辑。这使得系统更容易维护和扩展。
        
    
2. **高内聚，专注业务开发**
    
    - **解读**：程序员可以专注于某一层的开发。后端开发人员专注于 Model 的业务逻辑，前端开发人员专注于 View 的展示效果，分工明确，效率更高。
        
    
3. **代码复用性增强**
    
    - **解读**：一个 Model（如 `UserService`）可以被多个不同的 Controller 调用；一个数据模型（如 `User`对象）也可以被多种 View（如用户详情页、用户列表页）复用。
        
    
4. **代码可维护性增强**
    
    - **解读**：由于代码被组织在结构清晰的层中，当出现 bug 或需要增加新功能时，能够快速定位到相关代码并进行修改，而不至于“牵一发而动全身”。
        
    

### 四、 实际应用示例

以用户登录功能为例：

1. **用户**在浏览器输入用户名和密码，点击“登录”（**请求**到达）。
    
2. **Controller**（如 `LoginServlet`或 `LoginController`）接收到请求，获取用户名和密码参数。
    
3. **Controller**调用 **Model**（如 `UserService`的 `login`方法）进行身份验证。
    
4. **Model**执行业务逻辑：查询数据库，比对密码，返回登录结果（成功或失败）。
    
5. **Controller**根据 **Model**返回的结果，决定跳转到哪个 **View**：
    
    - 登录成功 -> 跳转到首页（`home.jsp`）。
        
    - 登录失败 -> 跳转回登录页并显示错误信息（`login.jsp`）。
        
    
6. **View**（JSP 页面）被渲染，生成最终的 HTML，返回给用户的浏览器展示。
