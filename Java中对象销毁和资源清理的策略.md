- ​**Java与C++的对比**​：Java没有显式析构器，因为**自动**垃圾回收处理**内存**回收；而C++需要手动析构器来释放内存。
    
- ​**非内存资源管理**​：对象可能使用文件、网络连接等**系统资源**，这些需要**显式清理**。推荐使用**close方法**（如实现AutoCloseable接口）来及时释放资源。
    
- ​**替代方案**​：对于延迟清理，可使用Runtime.addShutdownHook在JVM退出时执行清理，或使用Java 9的Cleaner类注册清理动作（当对象不可达时触发）。
    
- ​**finalize方法弃用**​：finalize方法原本用于垃圾回收前执行清理，但执行时间不确定，可能导致资源泄漏或性能问题，因此已弃用，不应使用。
    
- ​**实践建议**​：优先使用try-with-resources（第7章）确保资源关闭，避免依赖finalize。
